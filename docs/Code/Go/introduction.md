# Go 简介

Go 由 Google 发起，是一门静态类型、编译型、带垃圾回收和内建并发机制的通用语言。
本篇介绍 Go 的定位、运行模型、优势和边界，不把“语法简单”误解为“工程问题自动消失”。

## 语言定位

Go 主要解决大型软件开发中的构建速度、部署复杂度、协作一致性和并发服务编程问题。
它使用较小的语言核心配合统一工具链，使不同项目之间保持相近的代码风格。

常见使用场景包括：

- Web API、微服务和网关。
- 容器、云原生与基础设施工具。
- 命令行程序和跨平台运维工具。
- 网络代理、消息处理和并发任务服务。

Go 不直接提供类继承、运算符重载和宏系统。
需要复杂桌面 GUI、浏览器端运行或特定数值计算库时，应先评估生态，而不是只看语言性能。

## 核心特征

- 静态类型：大多数类型错误在编译阶段发现。
- 原生编译：通常生成可直接部署的本机可执行文件。
- 垃圾回收：自动管理大多数堆内存，但不管理文件、锁和网络连接的生命周期。
- goroutine：轻量并发执行单元，由 Go 运行时调度。
- channel：用于 goroutine 间通信和同步，但不是所有共享状态的唯一方案。
- 组合优先：通过结构体嵌入和小接口组合行为。
- 统一工具链：`go build`、`go test`、`go fmt` 和 `go vet` 随 SDK 提供。

## 最小程序

```go
package main

import "fmt"

func main() {
	fmt.Println("Go version baseline: 1.22+")
}
```

`package main` 声明可执行程序包，`main.main` 是程序入口。
`fmt.Println` 来自标准库 `fmt` 包；导入后未使用会导致编译错误。

## 编译与运行 API

常用命令如下：

```bash
go version
go env
go run .
go build .
go test ./...
go doc fmt.Println
```

`go run` 面向快速运行，`go build` 面向构建产物，`go test` 负责测试包。
模块由 `go.mod` 描述，现代项目应使用 Go Modules 管理依赖。

## 差异示例一：goroutine 不是操作系统线程

普通调用会等待函数返回：

```go
work()
fmt.Println("work finished")
```

加入 `go` 后，调用并发执行，当前函数不会自动等待：

```go
go work()
fmt.Println("work started")
```

第二段必须使用 `sync.WaitGroup`、channel 或其他同步机制管理完成条件。
不能依赖 `time.Sleep` 猜测 goroutine 已完成。

## 差异示例二：错误值不是自动抛出的异常

```go
data, err := os.ReadFile("config.json")
if err != nil {
	return fmt.Errorf("read config: %w", err)
}
fmt.Println(len(data))
```

`os.ReadFile` 返回错误值，调用者决定记录、包装、重试或返回。
`panic` 适合不可恢复的程序不变量破坏，不应替代普通错误处理。

## 值语义与引用语义边界

Go 的赋值和传参都是值复制。
切片、映射、channel、函数和接口值内部可能引用共享数据，因此复制描述符后仍可能观察到共同状态。
指针可以显式共享某个变量，但 Go 不支持指针算术。

```go
func increment(n int) {
	n++
}

func incrementAt(n *int) {
	(*n)++
}
```

第一个函数只修改参数副本，第二个函数通过指针修改调用者变量。

## 常见错误

- 认为有垃圾回收就不需要关闭文件、响应体或数据库结果集。
- 认为 goroutine 很轻量就可以无限创建，忽视退出和背压。
- 用空接口 `any` 逃避类型设计，导致运行时断言遍布代码。
- 为模仿面向对象继承建立复杂嵌入层次，降低可读性。
- 把编译成功等同于无数据竞争、无死锁或无业务错误。

## 工程实践

- 优先使用标准库和小接口，依赖应有明确收益。
- 包名简短并表达职责，不使用 `util`、`common` 堆放无关代码。
- 接口通常由使用方定义，只包含调用方真正需要的方法。
- 并发前先写正确的同步版本，再通过测量决定是否并发化。
- 使用 `go test -race ./...` 检查测试路径上的数据竞争。
- 通过 `go.mod` 固定模块语义，通过 CI 统一检查命令。

## 版本边界

本教程建议 Go 1.22+。
Go 1.22 修正了 `for` 循环变量在每次迭代中的语义，并支持整数 `range`；旧版本代码在闭包捕获循环变量时可能表现不同。
若项目必须兼容更早版本，应以 `go.mod` 的 `go` 指令和对应版本发布说明为准。

Go 的优势来自约束和一致性。理解这些约束的适用边界，比记忆“Go 很快”或“Go 天生适合并发”更重要。
