# Go 语言变量作用域

作用域决定标识符在源码中的可见范围，生命周期决定对应变量在运行时存在多久。
二者相关但不相同：不可见的局部变量仍可能因指针或闭包引用而继续存活。

## 作用域层级

Go 常见词法作用域包括：

- 预声明标识符所在的 universe block。
- 包作用域。
- 文件作用域，主要影响导入名称。
- 函数作用域和函数体块。
- `if`、`switch`、`for`、`select` 等隐式或显式语句块。
- 内层花括号创建的嵌套块。

标识符解析通常选择当前点可见的最内层声明。

## 包作用域

```go
package config

const DefaultPort = 8080

var loaded bool

func Load() {
	loaded = true
}
```

同一包的其他普通源文件可以使用 `DefaultPort` 和 `loaded`。
首字母大小写只影响包外导出，不影响同包文件之间访问。

包级变量生命周期通常贯穿进程，必须考虑初始化、测试隔离和并发访问。

## 文件作用域导入

导入名称只在声明它的源文件中可见：

```go
package report

import "fmt"

func Print() {
	fmt.Println("report")
}
```

同包另一个文件若也使用 `fmt`，必须自行导入。
包的其他顶层声明则在整个包作用域可见，这体现了导入与普通包级名称的差异。

## 函数参数与局部变量

```go
func greet(name string) string {
	prefix := "hello"
	return prefix + ", " + name
}
```

参数 `name` 和局部变量 `prefix` 只能在函数体相关范围内使用。
函数返回后，若没有逃逸引用，存储可被回收或复用；具体栈堆位置是编译器实现决策。

## 块作用域

```go
value := 10
{
	value := 20
	fmt.Println(value)
}
fmt.Println(value)
```

内层 `value` 遮蔽外层名称，输出 20 和 10。
遮蔽是合法语法，但可能让错误处理和状态更新偏离预期。

## `if` 初始化作用域

```go
if value, err := strconv.Atoi(input); err != nil {
	return err
} else {
	fmt.Println(value)
}
```

`value` 和 `err` 从初始化语句开始，在条件、所有分支中可见，离开整个 if 后不可见。
若成功结果要在分支后继续使用，应在外层声明并使用赋值。

## `for` 与 `range` 作用域

```go
for i := 0; i < 3; i++ {
	fmt.Println(i)
}
// i 在这里不可见
```

Go 1.22 起，每次迭代声明的循环变量采用新的迭代变量语义。
这改善闭包捕获行为，但共享外部变量仍需同步，旧模块版本也可能保留旧语义。

## 闭包与生命周期

```go
func makeCounter() func() int {
	count := 0
	return func() int {
		count++
		return count
	}
}
```

`makeCounter` 返回后，`count` 已离开普通源码可见范围，但闭包仍引用它，因此其生命周期继续。
作用域是编译期名称规则，生命周期由实际引用和实现决定。

## 预声明标识符遮蔽

```go
len := 10
fmt.Println(len)
// fmt.Println(len("abc")) // len 已被变量遮蔽，无法调用内建函数
```

`len`、`cap`、`make`、`new`、`error` 等不是关键字，可以被重新声明。
除极小且无歧义的范围外，不应遮蔽这些常用名称。

## 差异示例一：`:=` 与 `=` 对作用域的影响

```go
result := 0
if ok {
	result := compute()
	fmt.Println(result)
}
fmt.Println(result)
```

`:=` 创建内层变量，外层仍为 0。
改为 `result = compute()` 才会更新外层变量。

## 差异示例二：包作用域与文件导入作用域

一个文件声明的包级函数可被同包其他文件直接调用。
一个文件导入的 `json` 名称不能被同包其他文件共享，每个使用它的文件都必须导入。
因此删除某文件看似“未使用”的导入，可能只影响该文件而不会改变包级 API。

## 差异示例三：作用域与生命周期

返回局部变量地址是安全的：

```go
func newCount() *int {
	count := 0
	return &count
}
```

`count` 的名称在函数外不可见，但返回指针使变量继续存在。
这不意味着所有局部变量都分配在堆上；编译器通过逃逸分析决定存储位置。

## 常见错误

- 在 `if err := ...` 内遮蔽外层 `err`，函数最终返回了错误的值。
- 包级变量与局部变量同名，读者无法快速判断修改的是哪一份状态。
- 假设同包一个文件的导入可供其他文件使用。
- 把作用域离开误认为对象必定立即释放。
- 闭包捕获可变变量后跨 goroutine 使用，造成数据竞争。
- 遮蔽 `len`、`error`、`context` 等常见名称，降低可读性。

## 工程实践

- 使用最小必要作用域，临时变量尽量靠近使用位置。
- 对 `:=` 保持警惕，代码审查重点检查错误和结果变量遮蔽。
- 减少包级可变状态，通过构造参数显式传递依赖。
- 闭包捕获的共享状态注明同步和所有权策略。
- 不根据“栈变量/堆变量”猜优化，使用编译器诊断和基准验证。
- 大函数拆分为职责明确的小函数，降低同名变量和复杂作用域风险。

掌握作用域后，应能解释一个名称指向哪条声明、何处可见，以及变量为何可能在离开源码块后仍然存活。
