# HTML ASCII 参考

ASCII（American Standard Code for Information Interchange）定义 0 至 127 的字符。
它是 Unicode 和 UTF-8 的兼容子集，但现代 HTML 文档仍应明确使用 UTF-8。

## 控制字符

| 十进制 | 十六进制 | 缩写 | 含义 |
| ---: | ---: | --- | --- |
| 0 | `00` | NUL | 空字符，不应出现在普通 HTML 文本中 |
| 8 | `08` | BS | 退格 |
| 9 | `09` | HT | 水平制表符 |
| 10 | `0A` | LF | 换行，HTML 源码常用行结束符 |
| 12 | `0C` | FF | 换页 |
| 13 | `0D` | CR | 回车，常与 LF 组成 CRLF |
| 27 | `1B` | ESC | 转义字符 |
| 32 | `20` | SP | 普通空格 |
| 127 | `7F` | DEL | 删除控制字符 |

除空格、制表和换行外，控制字符通常不适合直接写入 HTML。

## 标点与数字

| 十进制 | 字符 | HTML 中的常见含义 |
| ---: | :---: | --- |
| 33 | `!` | 感叹号 |
| 34 | `"` | 属性值常用引号，可写作 `&quot;` |
| 35 | `#` | URL 片段标识符开头 |
| 37 | `%` | URL 百分号编码开头 |
| 38 | `&` | 字符引用开头，应按上下文写作 `&amp;` |
| 39 | `'` | 单引号，可写作 `&#39;` |
| 40–41 | `(` `)` | 圆括号 |
| 42 | `*` | 星号 |
| 43 | `+` | 加号；表单编码中可能表示空格 |
| 44 | `,` | 逗号 |
| 45 | `-` | 连字符，常用于文件名和属性值 |
| 46 | `.` | 句点、扩展名或相对路径 |
| 47 | `/` | URL 路径分隔符、结束标签前缀 |
| 48–57 | `0`–`9` | 十进制数字 |
| 58 | `:` | URL 方案或端口分隔符 |
| 59 | `;` | 字符引用结尾 |
| 60 | `<` | 标签开头；文本中写作 `&lt;` |
| 61 | `=` | 属性名与值的分隔符 |
| 62 | `>` | 标签结尾；需要时写作 `&gt;` |
| 63 | `?` | URL 查询部分开头 |
| 64 | `@` | 常见于邮箱地址 |

## 拉丁字母与其余符号

| 十进制范围 | 字符范围 | 说明 |
| --- | --- | --- |
| 65–90 | `A`–`Z` | ASCII 大写拉丁字母 |
| 91–96 | 六个 ASCII 标点 | 方括号、反斜杠、脱字符、下划线、反引号 |
| 97–122 | `a`–`z` | ASCII 小写拉丁字母 |
| 123–126 | 四个 ASCII 标点 | 花括号、竖线、波浪号 |

HTML 标签名按 ASCII 不区分大小写，但推荐统一使用小写。
URL、文件名和服务器路径是否区分大小写取决于协议、平台与服务器配置。

## 与 Unicode、UTF-8 的关系

| 概念 | 范围或性质 | 使用建议 |
| --- | --- | --- |
| ASCII | 128 个码位，`U+0000`–`U+007F` | 适合协议标记和基础语法 |
| ISO-8859-1 | 传统单字节字符集 | 不作为新 HTML 的默认选择 |
| Unicode | 字符集合与码位标准 | 文本处理的现代基础 |
| UTF-8 | Unicode 的可变长字节编码 | 现代 HTML 默认选择 |

同一个 ASCII 字符在 ASCII、ISO-8859-1 和 UTF-8 中具有相同的单字节值。
中文、货币符号和大多数国际文字超出 ASCII，应直接保存为 UTF-8。

## HTML 使用要点

- 文档使用 `<meta charset="utf-8">`，服务器也发送 UTF-8 `Content-Type`。
- 不用 ASCII 数字范围判断所有 Unicode 字母、数字或用户名合法性。
- 显示 HTML 语法字符时优先使用字符引用，参见[字符实体](entities.md)。
- URL 中是否编码由字符所处组件决定，参见[URL 编码](url-encoding.md)。
- 不把不可见控制字符混入标识符、日志或安全敏感值。
