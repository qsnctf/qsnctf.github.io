# HTML ISO-8859-1 参考

ISO-8859-1（Latin-1）是历史上的单字节字符集，覆盖 `0x00` 至 `0xFF`。
新 HTML 文档应使用 UTF-8；本页用于识别旧数据、旧声明和字符编号。

## 范围概览

| 十六进制 | 十进制 | 内容 |
| --- | ---: | --- |
| `00`–`1F` | 0–31 | C0 控制字符 |
| `20`–`7E` | 32–126 | 与 ASCII 相同的可打印字符 |
| `7F` | 127 | DEL 控制字符 |
| `80`–`9F` | 128–159 | C1 控制字符，不是可打印标点 |
| `A0`–`BF` | 160–191 | 不换行空格、货币与通用符号 |
| `C0`–`FF` | 192–255 | 西欧带附加符号的拉丁字母 |

浏览器历史兼容行为可能把标签为 ISO-8859-1 的网页按 Windows-1252 解码。
因此 `0x80`–`0x9F` 在旧网页中可能显示弯引号或欧元符号，但它们不属于 ISO-8859-1。

## `C0` 至 `FF` 字母组

| 范围 | 主要字符 | 说明 |
| --- | --- | --- |
| `C0`–`D6` | À 至 Ö | 大写重音拉丁字母 |
| `D7` | × | 乘号，不是字母 `x` |
| `D8`–`DE` | Ø 至 Þ | 其余大写拉丁字母 |
| `DF` | ß | 德语小写 sharp s |
| `E0`–`F6` | à 至 ö | 小写重音拉丁字母 |
| `F7` | ÷ | 除号 |
| `F8`–`FF` | ø 至 ÿ | 其余小写拉丁字母 |

## 编码迁移

| 情况 | 正确处理 |
| --- | --- |
| 已知源字节为 Latin-1 | 按 ISO-8859-1 解码，再以 UTF-8 编码保存 |
| 已知源字节为 Windows-1252 | 按 Windows-1252 解码，不要误标 Latin-1 |
| 只看到乱码文本 | 找回原始字节和实际编码，避免反复猜测转码 |
| 新建网页 | 编辑器、文件、模板和 HTTP 头统一使用 UTF-8 |

不要通过“把乱码再编码一次”盲目修复数据；错误转换可能永久丢失原字符。
字符引用表示 Unicode 字符，不会把整个文档切换为某种字节编码。
