## SHA-1

![](https://pic1.imgdb.cn/item/688493f258cb8da5c8e1c361.png)

**SHA-1 加密流程**

1. **填充消息（Padding）**

   - 原始消息会被填充，使得总长度是 512 的倍数

   - 会在消息后面加上 `1` bit，然后补 `0`，最后加上原始消息长度（64 bit 表示）

   - 这是为了让算法可以分块处理

2. **分块处理**

   - 将填充后的消息划分为多个 **512 位块**

   - 每个块将被拆分为 16 个 32-bit 的子块

3. **初始化常量（5 个 32-bit 寄存器）**

   - 使用 5 个固定的初始哈希值：

     ```ini
     h0 = 0x67452301
     h1 = 0xEFCDAB89
     h2 = 0x98BADCFE
     h3 = 0x10325476
     h4 = 0xC3D2E1F0
     ```

4. **主循环（80 次操作）**

   - 对每个 512-bit 块，执行 80 次的位运算和逻辑运算（如 AND、OR、XOR、移位、加法）：
     - 每 20 次分为一组，共 4 组；
     - 每组使用不同的逻辑函数 `f(t)` 和常量 `k(t)`

   - 每次迭代会更新 5 个哈希寄存器的值

5. **合并结果**
   - 最终，5 个哈希寄存器的值拼接起来，得到 160 位（20 字节）的哈希值，即 SHA-1 输出

**SHA-1 加密特点:**

```
不管输入多长，输出总是 160 位（20 字节）
```

**SHA-256 加密流程**

1. **准备消息**：
   - 把你的文本变成二进制（比如“abc”变成01100001 01100010 01100011）
   - 在后面加上 `1` 和一些 `0`，最后再加上“原来多长”的信息，让总长度变成512的倍数
2. **分块处理**：
   - 把数据每512位分成一块（一块就像一页纸），逐页处理
3. **初始化8个变量**：
   - 它有8个固定的“起始值”，类似一个记事本的第一页
4. **压缩循环（64轮）**：
   - 每一块数据要被处理64次，每次都根据一些规则不停“搅拌”这8个变量
5. **更新结果**：
   - 每轮计算出来的结果都加到原来的值里，最后形成一个唯一的“指纹”
6. **输出结果**：
   - 得到一个256位的二进制，通常显示为64位的十六进制字符串

**SHA-256 加密特点:**

```
无论输入多长，输出永远是 256 位（二进制），也就是 64 个十六进制字符
```
