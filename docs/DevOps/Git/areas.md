# Git 工作区、暂存区和版本库

Git 管理文件时，最重要的是理解三个区域：工作区、暂存区和版本库。

## 工作区

工作区是你实际看到和编辑的目录。你用编辑器修改的文件，都先出现在工作区。

查看工作区状态：

```bash
git status
```

## 暂存区

暂存区保存“准备提交”的改动。

```bash
git add README.md
git add .
```

暂存区让你可以控制每次提交包含哪些内容，而不是把所有修改一次性提交。

## 版本库

版本库位于 `.git` 目录中，保存提交历史、分支、标签等数据。

提交暂存区内容：

```bash
git commit -m "Update README"
```

## 文件状态

| 状态 | 含义 |
| ---- | ---- |
| Untracked | 新文件，Git 还没有跟踪 |
| Modified | 文件已修改，但未暂存 |
| Staged | 文件已暂存，等待提交 |
| Committed | 文件已提交到版本库 |

## 查看差异

查看工作区和暂存区差异：

```bash
git diff
```

查看暂存区和最新提交差异：

```bash
git diff --cached
```
