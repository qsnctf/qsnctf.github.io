# PNG宽高修改

在 PNG 文件的 `IHDR` 块中，图像的宽度和高度是作为无符号整数存储的，分别占用 4 字节。这意味着宽度和高度的范围是 0 到 4,294,967,295（$ 2^32 - 1$）像素。通过修改这些值，攻击者可以对图像的尺寸进行更改。

> PNG（Portable Network Graphics）是一种无损压缩的图像格式，广泛用于互联网图像传输。PNG 格式包含多个不同的块（Chunk），每个块都包含特定的信息。例如，`IHDR` 块包含图像的基本信息，如图像的宽度、高度、颜色深度、颜色类型等。

PNG 文件的结构大致如下：

- **文件头（8 字节）**：标识文件类型（`\x89PNG\r\n\x1a\n`）。
- **IHDR 块（13 字节）**：包含宽度、高度、位深度等基本信息。
- **IDAT 块**：包含实际的图像数据，经过压缩。
- **IEND 块**：文件的结束标记。

在 CTF 竞赛中，参赛者通常会尝试修改这些信息，特别是 `IHDR` 块中的宽度和高度，以达到某种特定目的。

#### 二、PNG 宽高修改

在 PNG 文件的 `IHDR` 块中，图像的宽度和高度是作为无符号整数存储的，分别占用 4 字节。这意味着宽度和高度的范围是 0 到 4,294,967,295（2^32 - 1）像素。通过修改这些值，攻击者可以对图像的尺寸进行更改。

##### 修改 PNG 宽高的步骤：

1. **定位 `IHDR` 块**：

   - `IHDR` 块紧跟在文件头之后，长度为 13 字节。这个块的格式如下：

     ```
     4 字节 - 块长度（IHDR 部分的字节数，为 13）
     4 字节 - 块类型（"IHDR"）
     4 字节 - 宽度（大端格式）
     4 字节 - 高度（大端格式）
     1 字节 - 位深度（如 8）
     1 字节 - 颜色类型（如 RGB）
     1 字节 - 压缩方法
     1 字节 - 过滤方法
     1 字节 - 限制方法
     ```

   - 可以使用工具（如 `hexedit` 或 Python 脚本）来定位并查看这些信息。

2. **修改宽高字段**：

   - 使用十六进制编辑器或编程语言（如 Python）修改文件中的宽度和高度字段。宽度和高度字段为 4 字节，按大端格式存储。修改时可以尝试调整为不同的值，以观察是否影响图像的显示效果。

   示例 Python 脚本，修改 PNG 文件宽高：

   ```python
   def modify_png_dimensions(png_file, new_width, new_height):
       with open(png_file, 'rb') as f:
           data = f.read()
   
       # 定位到 IHDR 块
       ihdr_offset = data.find(b'IHDR') + 4
       # 修改宽度和高度（大端格式）
       data = data[:ihdr_offset] + new_width.to_bytes(4, 'big') + new_height.to_bytes(4, 'big') + data[ihdr_offset + 8:]
   
       # 保存修改后的文件
       with open('modified_' + png_file, 'wb') as f:
           f.write(data)
   
   modify_png_dimensions('example.png', 800, 600)
   ```

3. **CRC32 校验与文件完整性**：

   - 修改 PNG 文件的内容后，必须确保图像的完整性。PNG 文件使用 CRC32 校验和来验证数据的完整性。如果在 `IHDR` 块或其他块中修改了内容，必须重新计算 CRC32 校验和，否则文件可能会被认为是损坏的。

   CRC32 校验和是用来确保文件数据在传输或存储过程中未被篡改。每个数据块（如 `IHDR`、`IDAT`）都包含 CRC32 校验和，用于验证该块数据的完整性。

#### 三、CRC32 爆破

在 CTF 中，修改 PNG 文件的宽高之后，可能需要破解或绕过 CRC32 校验，以确保修改后的文件不会被标记为损坏。CRC32 校验和是一个 32 位的哈希值，用于验证数据的完整性。

##### CRC32 爆破的步骤：

1. **理解 CRC32 校验**：

   - CRC32 是一种广泛使用的哈希算法，生成一个 32 位的校验值。对于 PNG 文件，每个数据块（如 `IHDR`、`IDAT`）都会有一个对应的 CRC32 校验和。修改任何内容（如图像的宽高）后，需要重新计算 CRC32 校验和。

2. **模拟 CRC32 校验计算**：

   - 计算 CRC32 校验和的原理相对简单。每个块的数据在进行 CRC32 计算时，必须包括数据本身以及块类型。修改 PNG 文件后，我们需要重新计算这些数据块的 CRC32 值并替换原有的 CRC32 校验和。

   使用 Python 的 `zlib` 库可以非常方便地计算 CRC32 校验和：

   ```python
   import zlib
   
   def calculate_crc32(data):
       return zlib.crc32(data) & 0xffffffff
   
   # 示例：计算 "IHDR" 块数据的 CRC32 校验和
   data = b'IHDR' + b'\x00\x00\x03\x20' + b'\x00\x00\x02\x58'  # 假设数据包含宽高
   crc32_value = calculate_crc32(data)
   print(f"CRC32: {crc32_value:08x}")
   ```

3. **暴力破解 CRC32**：

   - 在一些 CTF 题目中，如果没有提供正确的 CRC32 值，参赛者可以使用暴力破解技术来猜测正确的 CRC32 校验和。这通常涉及到对 CRC32 计算进行修改，并尝试多个不同的块内容，直到 CRC32 值匹配。
   - 对于简单的 CRC32 爆破问题，可能需要尝试多种组合，并利用计算机的计算能力来加速爆破过程。

4. **工具辅助**：

   - 除了手动编写脚本，常用的工具如 `pngcrush`、`pngcheck` 等可以帮助我们验证 PNG 文件的有效性或进行 CRC32 校验。对于复杂的图像编辑和 CRC32 计算，工具的使用往往能大大提高效率。
