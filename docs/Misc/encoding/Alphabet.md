# 字母表编码

## 什么是字母表编码？

字母表编码通常指把字母按字母表顺序转换为数字的编码方式。最常见的规则是：

```text
A = 1, B = 2, C = 3, ..., Z = 26
```

它也叫 A1Z26 编码。CTF 中经常用它来隐藏短文本、flag 片段或作为多层编码中的一层。

## 标准映射表

| 字母 | 数字 | 字母 | 数字 | 字母 | 数字 | 字母 | 数字 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| A | 1 | H | 8 | O | 15 | V | 22 |
| B | 2 | I | 9 | P | 16 | W | 23 |
| C | 3 | J | 10 | Q | 17 | X | 24 |
| D | 4 | K | 11 | R | 18 | Y | 25 |
| E | 5 | L | 12 | S | 19 | Z | 26 |
| F | 6 | M | 13 | T | 20 |  |  |
| G | 7 | N | 14 | U | 21 |  |  |

## 编码示例

字符串：

```text
CTF
```

逐字符转换：

| 字母 | C | T | F |
| ---- | - | - | - |
| 数字 | 3 | 20 | 6 |

所以：

```text
CTF -> 3 20 6
```

## 解码示例

密文：

```text
6 12 1 7
```

查表：

| 数字 | 6 | 12 | 1 | 7 |
| ---- | - | -- | - | - |
| 字母 | F | L | A | G |

得到：

```text
FLAG
```

## 常见变形

- 从 0 开始编号：`A = 0, B = 1, ..., Z = 25`。
- 倒序字母表：`A = 26, B = 25, ..., Z = 1`。
- 使用小写字母，但映射规则不变。
- 数字没有分隔符，需要按 `1-26` 的范围尝试切分。
- 与凯撒、ROT、Base、Hex 等编码组合出现。

## CTF 识别要点

- 密文由 `1-26` 范围内的数字组成。
- 分隔符可能是空格、逗号、短横线、斜杠。
- 无分隔数字串可能存在多种切分，需要结合 flag 格式判断。
- 如果结果不通顺，尝试 `A=0` 或倒序字母表。
