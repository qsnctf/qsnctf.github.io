# MCP 协议

MCP 可以在不同语境中指代不同协议或规范。近年来常见的 MCP 是 Model Context Protocol，用于让应用以标准方式连接模型、工具和数据源。

## 与传统网络协议的区别

传统网络协议通常定义传输层或应用层通信细节，例如端口、报文和连接状态。MCP 更偏向应用集成协议，关注模型应用如何发现工具、调用工具和读取上下文。

## 学习重点

- 客户端、服务端和工具的角色。
- 请求、响应和能力声明。
- 权限边界和数据暴露范围。
- 与底层 HTTP、stdio 或其他传输方式的关系。

## 安全注意

MCP 类协议要重点关注权限最小化、工具调用审计、敏感数据隔离和提示注入风险。本站也有独立的 [MCP](../../../MCP/index.md) 栏目。
