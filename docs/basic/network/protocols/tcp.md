# TCP 协议

TCP（Transmission Control Protocol）是面向连接的可靠字节流协议。

## 核心能力

- 三次握手建立连接。
- 序列号和确认号保证有序传输。
- 丢包重传。
- 流量控制和拥塞控制。
- 四元组标识连接。

## 三次握手

```text
客户端 -> SYN -> 服务端
客户端 <- SYN, ACK <- 服务端
客户端 -> ACK -> 服务端
```

握手完成后，双方可以传输字节流。

## 常见标志位

| 标志 | 含义 |
| --- | --- |
| SYN | 建立连接 |
| ACK | 确认 |
| FIN | 正常关闭 |
| RST | 重置连接 |
| PSH | 尽快交给应用 |

## 分析提示

TCP 是字节流，不保留应用层消息边界。应用协议必须自己定义长度、分隔符或关闭连接作为边界。
